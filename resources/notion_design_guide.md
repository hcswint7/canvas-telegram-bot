# Complete Notion Database Design Guide for AI Agents

(Archived from user input for future reference)

**Core Philosophy:** Structure serves the user. Prioritize visual clarity, reduced cognitive load, and actionability.

**Visual Design:**
- Use semantic colors (Red = Urgent, Yellow = In Progress, Green = Done, Blue = Info).
- Use Emojis on every page icon as visual anchors.
- Use Cover Images.
- Use Callout blocks for critical info.

**Database Architecture:**
- Property Order: Title -> Status -> Assignee -> Due Date -> Priority -> Tags
- Recommended Task DB Properties: Title, Status, Priority, Due Date, Project, Assignee, Tags, Estimated Time, Energy Level (High Focus, Medium, Low Energy).

**View Architecture:**
- All Tasks (Table)
- Today (Filtered Table)
- In Progress (Board)
- Schedule (Calendar)

**Dashboard Layout:**
- Use columns.
- Callout blocks with emojis for section dividers.
- Keep the homepage to one screen height.
- Synced blocks for shared status updates.
